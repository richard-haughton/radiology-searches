## Security Requirements

**IMPORTANT**: You must update Firestore security rules to enable this feature.

Add the following rules for the `/sharedPatterns/{docId}` collection:

```javascript
match /sharedPatterns/{docId} {
  // Read: Anyone authenticated
  allow read: if request.auth != null;
  
  // Create: Authenticated users only
  allow create: if request.auth != null &&
                   request.resource.data.authorId == request.auth.uid &&
                   request.resource.data.patternId != null &&
                   request.resource.data.patternName != null &&
                   request.resource.data.modality != null &&
                   request.resource.data.sharedAt != null &&
                   request.resource.data.importCount == 0;
  
  // Update: Only owner, importCount only
  allow update: if request.auth != null &&
                   resource.data.authorId == request.auth.uid &&
                   request.resource.data.diff(resource.data)
                      .affectedKeys().hasOnly(['importCount']);
  
  // Delete: Only owner
  allow delete: if request.auth != null &&
                   resource.data.authorId == request.auth.uid;
}
```

## Known Limitations

1. **Firestore Rules Required**: Feature won't work without proper security rules
2. **No Moderation**: Any authenticated user can share patterns (consider adding moderation for production)
3. **No Direct Messaging**: No way for users to contact pattern authors
4. **No Ratings**: Users can't rate or review patterns
5. **No Versioning**: Importing creates a copy; edits don't update the original
6. **Single Copy**: Users can't have multiple versions of the same shared pattern
